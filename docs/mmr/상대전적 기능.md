# 상대전적 (H2H) 기능

> 두 유저가 같은 내전에 있었던 기록을 모아 비교 분석하는 기능 — 백엔드 단일 정리 문서.
> 데이터 원천: `mmr_participant_metric` (경기·유저당 1행). 적재·backfill은 `metric_implementation_guide.md` 참고.

## 1. 개요

| 모드 | 정의 | 판별 |
|---|---|---|
| **맞붙은 (against)** | 같은 게임, 다른 팀 | `a.game_team <> b.game_team` |
| **함께한 (together)** | 같은 게임, 같은 팀 | `a.game_team = b.game_team` |

- **API 2개**: 자주 만난 상대(`/frequent`) + 상세(`/`). 모드 토글·맞라인 토글은 재호출 없이 프론트 필터(상세 응답에 against·together 모두 포함).
- **식별**: 입력은 `riotName`/`riotNameTag`(기존 닉네임 검색), 내부 집계는 **본계정 병합된 `player_code`** 기준.

---

## 2. 데이터 원천 — `mmr_participant_metric`

- 한 row = 한 경기의 한 참가자. raw 49 + 파생 14 + 식별/파이프라인. 자연키 `(custom_match_id, puuid)`.
- **`player_code`**: 본계정 병합 식별자 (`match_participant.player_code`와 동일 규칙 — 부계정이면 본계정 playerCode). 적재 시 `puuidToPlayerCodeMap`으로 채우고, 부계정 연결/게임 삭제 시 동기화. H2H 식별·집계는 이 컬럼 기준.
- 적재·backfill 상세: `metric_implementation_guide.md` 참고.

**집계 베이스 (self join)**: 같은 `custom_match_id`에서 me(`a`) · oppo(`b`)를 player_code로 셀프 조인. `a.game_team <> b.game_team` → 맞붙은, `=` → 함께한. season·is_deleted 필터, played_date ASC(스트릭용).

## 3. API

### 3.1 자주 만난 상대 — `GET /api/h2h/:guildId/frequent`

기준 유저가 **맞붙은(다른 팀)** 상대를 본계정 단위로 집계. **시즌 기준**, matchups 내림차순.

| 파라미터 | 설명 |
|---|---|
| `riotName` (필수) · `riotNameTag` | 기준 유저 |
| `q` | 상대 `riotName#tag` 부분일치 필터 (검색 자동완성 겸용) |
| `season` | 미입력 현재 시즌(LOL_SEASON), `all` 전체 |
| `limit` | 기본 10, 최대 50 |

```jsonc
[ { "puuid":"...", "riotName":"구마유시", "riotNameTag":"T1",
    "mainLane":"ADC", "matchups":18, "winRate":66.7, "lastPlayedDate":"..." } ]
```
- **활성 메인 멤버만** 노출 (`guild_member` isMain·status='1'·is_deleted=false).
- 동명이인 → 단일 결과 대신 후보 목록 반환(`status:'error'`, `data: candidates[]`).

### 3.2 상대전적 상세 — `GET /api/h2h/:guildId`

두 유저의 프로필·메타 + `against` + `together`를 **단일 응답**으로.

| 파라미터 | 설명 |
|---|---|
| `riotName1`·`riotNameTag1` / `riotName2`·`riotNameTag2` | 두 유저 |
| `season` | 미입력 현재 시즌, `all` 전체 |
| `recentLimit`·`recentOffset` | 최근 맞대결 페이지네이션 (기본 6) |

```jsonc
{
  "me":   { "puuid","riotName","riotNameTag", "mmr":null, "mostLane","seasonWR" },
  "oppo": { /* 동일 */ },
  "totalMet": 83, "firstMet":"...", "lastMet":"...",
  "against":  { /* §3.3 */ },
  "together": { /* §3.4 */ }
}
```
- `mmr`: 길드 MMR 테이블 미존재 → 항상 `null`.
- `mostLane`·`seasonWR`: 해당 유저의 시즌 전체 기준.

### 3.3 `against` 블록

```jsonc
{
  "games","wins","losses","winRate",
  "streak": ["W","L",...],            // 전체 맞대결 시간순(오래된→최신)
  "seasonBreaks": [{ "index","label" }],   // season=all일 때만
  "mine":  { kda, dpm, laneGoldDiff, tdBefore15, turretPlates, expPerMin, deadTimePct, jungleCsEnemy },
  "oppos": { /* 동일 */ },             // 둘 다 JUG 매치업일 때만 jungleCsEnemy, 그 외 null
  "laneMatrix": { "TOP":{ "TOP":{c,w},... }, ... },  // 5×5 (빈 셀 0)
  "topLane": { "lane","count","wins" } | null,       // 가장 많이 맞붙은 동일 라인 (대각선 max)
  "matchups": [{ myLane, oppoLane, myChamp, oppoChamp, count, wins, myKda, kdaDiff }],  // count DESC, champion_id NULL 제외
  "insights": [ /* §3.5 */ ],
  "recent":   [ /* §3.6 */ ],
  "recentTotal": 39
}
```
- 지표 평균은 `AVG(파생컬럼)`, NULL 전부면 키 null.
- `kdaDiff` = 매치업 평균 KDA − 내 시즌 평균 KDA (부호 포함 문자열).

### 3.4 `together` 블록

```jsonc
{
  "games","wins","losses","winRate","streak",
  "laneCombos":   [{ mine, oppo, count, wins }],          // count DESC
  "topLaneCombo": { mine, oppo, count, wins } | null,     // = laneCombos[0]
  "duoChamps":    [{ mine, oppo, mineLane, oppoLane, count, wins, comboKda }],  // count DESC, champion_id NULL 제외
  "recent":       [ /* detail 없음 */ ]
}
```
- `comboKda` = 두 사람 (K+A) 합 / D 합(0이면 1).
- 지표 비교·라인 매트릭스·인사이트는 함께한 모드엔 **없음**.

### 3.5 인사이트 (against 전용, 최대 4장)

백엔드는 **`kind`(스타일) + `type`(종류) + raw 수치**만 전달, 한국어 문구·조사는 프론트가 조립.

| type | 조건 | 데이터 |
|---|---|---|
| `counterPick` (best) | 매치업 count≥2 & 승률≥60% | myChamp, oppoChamp, wins, losses, winRate, kdaDiff |
| `nemesis` (worst) | count≥2 & 승률≤40% | 동일 |
| `laneVsResult` (counter/best) | 패배≥3 & 라인골드우위 비율≥50% (반대면 best) | direction, total, laneCount |
| `momentum` (info) | 맞대결≥8 & 통산vs최근5판 ±20%p | direction, recentN, recentWins, recentWinRate, careerWinRate |
| `streak` (best/worst) | 최장 연승/패≥3 & A1~A4가 4장 미달 시 | streakKind, length, fromDate, toDate, currentLength |

### 3.6 최근 맞대결 + 세부보기

```jsonc
{ "matchId","playedDate","myResult","myLane","oppoLane","myChamp","oppoChamp",
  "myKda":"5/2/14","oppoKda":"3/6/8","gameLen":2088,
  "detail": { "mine": { /* 23키 */ }, "oppo": { /* 동일 */ } } }   // ← 맞라인 against 경기만
```
- `detail` 포함 조건: **against 모드 AND 맞라인(`myLane===oppoLane`)**. 미달이면 키 생략.
- 단일 경기 원시치, **NULL은 0으로** (프론트 숫자 가정). cs·epicKills·healShield는 COALESCE(0) 후 합산.
- 백엔드는 23키 전부 채워서 내려주고, 프론트가 `myLane` 기준으로 8칸을 골라 표시:

| detail 키 | raw 컬럼 / 공식 | 방향 |
|---|---|---|
| `dmg` | damage_to_champions | high |
| `taken` | damage_taken | ↓ low |
| `selfMit` | damage_self_mitigated | high |
| `gold` | gold_earned | high |
| `cs` | minions_killed + neutral_minions_killed | high |
| `kda` | (kills+assists)/max(deaths,1) | high |
| `laneGoldDiff` | lane_gold_diff (±) | high |
| `td15` | takedowns_before_15min | high |
| `underTurretTd` | takedowns_under_turret | high |
| `turretTd` | turret_takedowns | high |
| `plates` | turret_plates_destroyed | high |
| `enemyJungleCs` | jungle_cs_enemy | high |
| `objDmg` | damage_to_objectives | high |
| `epicKills` | dragon_kills + baron_kills + herald_kills | high |
| `objSteals` | objectives_stolen | high |
| `deadPct` | dead_time_pct | ↓ low |
| `vision` | vision_score | high |
| `wardsP` | wards_placed | high |
| `wardsK` | wards_killed | high |
| `controlW` | control_wards_bought | high |
| `healShield` | heal_on_teammates + shield_on_teammates | high |
| `ccTime` | cc_time | high |
| `missPings` | enemy_missing_pings | high |

**라인별 표시 8칸 (프론트가 `myLane`으로 선택)**

| 라인 | 8칸 |
|---|---|
| TOP | dmg · taken · selfMit · laneGoldDiff · td15 · underTurretTd · turretTd · ccTime |
| JUG | dmg · gold · kda · enemyJungleCs · objDmg · epicKills · td15 · objSteals |
| MID | dmg · taken · gold · cs · laneGoldDiff · kda · td15 · turretTd |
| ADC | dmg · taken · gold · cs · laneGoldDiff · plates · deadPct · turretTd |
| SUP | vision · wardsP · wardsK · controlW · healShield · ccTime · missPings · td15 |

## 4. 확정 결정

- **본계정 병합**: `player_code` 컬럼에 적재 시 병합 저장 (match_participant와 동일). H2H 쿼리는 player_code 기준.
- **frequent 후보**: 활성 메인 멤버만.
- **season 기본**: 현재 시즌(LOL_SEASON), `all` 전체.
- **mmr**: 테이블 미존재 → 항상 null.
- **is_mmr_eligible 미필터**: 상대전적은 전 경기 포함 (MMR 계산과 정책 분리).
- **인사이트**: 백엔드 데이터만, 문구·조사는 프론트.
- **topLane / topLaneCombo**: 프론트 파생 부담 줄이려 백엔드가 미리 제공.
